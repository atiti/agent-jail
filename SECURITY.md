# Security Policy

## Supported scope

This project is intended to reduce accidental damage and improve operator control for agent-driven command execution. It is not a formal security sandbox or a guarantee of isolation.

## Reporting a vulnerability

Please do not open public issues for sensitive vulnerabilities.

When reporting, include:

- affected version or commit
- host OS and backend
- reproduction steps
- impact assessment

## Disclosure guidelines

- Avoid posting live credentials, hostnames, tokens, or operator-specific inventory.
- Prefer minimal reproductions over full private environment dumps.
- Treat bypasses of the broker, delegate boundary, policy engine, or backend containment as security-relevant.
